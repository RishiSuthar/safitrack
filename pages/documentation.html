# Quick Install

SafiTrack is a Progressive Web App (PWA). Field reps can install it from any modern browser on Android or iOS with no app store required. For backend integrations, use one of our official SDKs.

1
Create your organization
Sign up at app.safitrack.io and complete the onboarding wizard. You'll be assigned an Organization ID (org_…) and a default admin account.
2
Install the SDK
Add the SafiTrack SDK to your project using your preferred package manager.
3
Initialize & authenticate
Instantiate the client with your API key and preferred region. All requests are automatically signed.
4
Invite your team
Go to Settings → Team Members and send invitations. Field reps receive a magic-link install prompt for the PWA.
npm
yarn
pnpm
bash
# Install the core SDK
npm install @safitrack/sdk-core

# Optional: install analytics plugin
npm install @safitrack/plugin-analytics
bash
yarn add @safitrack/sdk-core
yarn add @safitrack/plugin-analytics # optional
bash
pnpm add @safitrack/sdk-core
javascript
import { SafiTrack } from '@safitrack/sdk-core';

const safi = new SafiTrack({
  apiKey:  'st_live_xxxxxxxxxxxxxxxxxxxx',
  region:  'africa-west-1',   // africa-west-1 | eu-central-1 | us-east-1
  timeout: 8000,
});

// Quick smoke test
const org = await safi.org.get();
console.log(`Connected to: ${org.name}`);
⚠️
Security reminder Never expose st_live_… keys in client-side or public code. Use environment variables and server-side proxies. For browser usage, generate short-lived session tokens via POST /auth/session.

# Authentication

SafiTrack uses API key authentication for server-to-server calls and short-lived JWT session tokens for client-side access. All requests must be made over HTTPS.

API Key types

Key prefix Environment Use case
st_live_… Production Real data, billable actions
st_test_… Sandbox Development & CI pipelines
st_sess_… Both Short-lived client tokens (TTL 4h)

Authorization header

http 
GET /v1/visits HTTP/1.1
Host: api.safitrack.io
Authorization: Bearer st_live_xxxxxxxxxxxxxxxxxxxx
Content-Type: application/json

Session tokens (for PWA / browser)

javascript
// Server-side: exchange API key for a session token
const { token } = await safi.auth.createSession({
  userId:     'usr_abc123',
  expiresIn:  14400,  // seconds
  scopes:     ['visits:write', 'companies:read'],
});

// Pass token to your frontend — never the raw API key
return res.json({ sessionToken: token });
💡
Tip — rotate keys automatically Use the POST /v1/keys/rotate endpoint in your CI/CD pipeline to rotate production keys on every deployment without downtime.

Request signing parameters

Parameter Type Required Description
api_key string required Your organization API key.
env enum required production or sandbox
idempotency_key string optional UUID to safely retry POST requests without duplicate records.
x-safi-version string optional Pin to an API version e.g. 2025-01-01. Defaults to latest.

# Core Concepts

SafiTrack is built around four primary entities. Every feature — from visit logging to territory assignment — builds on top of these primitives.

🏢
Companies
Professional entities — shops, clinics, distributors — that your team visits or services.
👤
People
Individual contacts within companies. Linked to one company but visible across your CRM.
📍
Visits
Geolocation-verified check-ins. Timestamped, auditable, and enriched with form data.
📦
Products
Inventory items tracked per company. Updated at the point of sale or delivery.

Relationships

A User (field rep) belongs to one or more Territories. Each territory contains Companies. Companies have People, pending Visits, and Inventory records. This hierarchy drives dashboards, performance scores, and access control.

json — entity relationships
{
  "organization": {
    "territories": [
      {
        "id": "ter_nairobi_cbd",
        "assignedReps": ["usr_amara", "usr_kwame"],
        "companies": [
          {
            "id": "cmp_001",
            "name": "Westside Pharmacy",
            "contacts": ["ppl_jane"],
            "lastVisit": "2026-01-18T09:42:00Z"
          }
        ]
      }
    ]
  }
}

# Field Sales Tracking

SafiTrack's visit engine is the heart of the platform. It combines GPS verification, custom data forms, and offline sync so your reps can work reliably in any connectivity environment.

Logging a visit

javascript
const visit = await safi.visits.create({
  companyId:  'cmp_001',
  repId:      'usr_amara',
  coords:     { lat: -1.2921, lng: 36.8219 },
  accuracy:   12,             // meters
  formData: {
    outcome:        'sale_made',
    orderValue:     4800,      // KES
    productsOrdered: ['sku_panadol_24', 'sku_ors_10pk'],
    notes:          'Stock was low — pushed order for next week',
  },
});

console.log(visit.id);  // vis_xxxxxxxx
💡
Geofencing Enable geofence enforcement in Settings → Visit Policy to reject check-ins logged more than N metres from the company's registered location. Default tolerance is 200 m.

Visit states

State Description
pending Saved offline, not yet synced to the server.
submitted Received by the API, pending geo-verification.
verified GPS coordinates confirmed within tolerance.
flagged Outside geofence or suspicious pattern detected — needs manager review.
rejected Marked invalid after review. Excluded from performance metrics.

Offline sync

The PWA caches visits locally using IndexedDB when the device is offline. Once connectivity is restored, the sync engine submits queued visits in chronological order and reconciles any conflicts using the server timestamp as the source of truth.

javascript
// Force a manual sync (usually automatic)
await safi.sync.flush();

// Check the offline queue length
const { pending } = await safi.sync.status();
console.log(`${pending} visits waiting to sync`);

# Territory Management

Territories define which companies each rep is responsible for. They can be drawn as geographic polygons on the map, defined by a list of company IDs, or auto-generated using our clustering algorithm.

Creating a territory

javascript
const territory = await safi.territories.create({
  name:    'Nairobi CBD',
  color:   '#2f5fd0',
  polygon: [                          // GeoJSON coordinates
    [36.8109, -1.2789],
    [36.8297, -1.2789],
    [36.8297, -1.3040],
    [36.8109, -1.3040],
  ],
  assignedReps: ['usr_amara', 'usr_kwame'],
});

Auto-cluster territories

The clustering endpoint groups all unassigned companies into N territories using K-Means distance weighting. Useful during initial setup.

http 
POST /v1/territories/cluster
{
  "targetCount": 8,
  "balanceByVisitLoad": true
}
💡
Re-balance tip Run the cluster endpoint quarterly to account for new companies added and rep turnover. Clustering is non-destructive — it returns a preview that you can accept or discard.

# Real-time Inventory

Track product stock levels at each company location. Inventory is updated automatically when a rep logs a visit with order or delivery data, or via direct API writes for warehousing integrations.

Recording stock

javascript
await safi.inventory.update({
  companyId: 'cmp_001',
  items: [
    { sku: 'sku_panadol_24', qty: 48,  action: 'set'   },
    { sku: 'sku_ors_10pk',   qty: 12,  action: 'add'   },
    { sku: 'sku_expired_x',  qty: 5,   action: 'remove' },
  ],
  repId:     'usr_amara',
  visitId:   'vis_abc789',  // link to the visit that triggered this
});

Low-stock alerts

Configure automatic alerts when a SKU falls below a threshold. Alerts are delivered via the notification system (see Notifications) and are also surfaced in the manager dashboard.

http 
POST /v1/inventory/alerts
{
  "sku":       "sku_panadol_24",
  "threshold": 10,
  "channels":  ["email", "in_app"],
  "assignee":  "usr_manager_01"
}

# Notifications

SafiTrack sends notifications via in-app, email, and SMS channels. Configure rules per organization, per team, or per individual rep.

Event Default channel Configurable?
visit.flagged in_app + email (manager) Yes
visit.verified in_app (rep) Yes
inventory.low_stock email Yes
rep.offline_too_long in_app (manager) Yes
report.ready email No
javascript
// Override notification channels for a user
await safi.notifications.setPreferences({
  userId:   'usr_amara',
  channels: {
    'visit.flagged': ['sms', 'in_app'],
    'visit.verified': [],   // disable
  },
});

# API Endpoints

Base URL: https://api.safitrack.io/v1. All responses are JSON. Dates follow ISO 8601. Pagination uses cursor-based after / before query params.

PATCH /companies/:id Update company fields.

GET /visits List visits with filters.
POST /visits Log a new visit.
GET /visits/:id Get visit detail.
POST /territories Create a territory.
GET /territories List all territories.

# Webhooks

Subscribe to real-time events by registering a webhook endpoint. SafiTrack signs every payload with an HMAC-SHA256 signature using your webhook secret.

Registering a webhook

javascript
await safi.webhooks.create({
  url:    'https://yourapp.com/webhooks/safitrack',
  events: ['visit.verified', 'visit.flagged', 'inventory.low_stock'],
  secret: process.env.WEBHOOK_SECRET,
});

Verifying signatures

javascript
import crypto from 'node:crypto';

function verifyWebhook(rawBody, signature, secret) {
  const expected = crypto
    .createHmac('sha256', secret)
    .update(rawBody)
    .digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(`sha256=${expected}`),
    Buffer.from(signature)
  );
}
⚠️
Respond quickly SafiTrack expects a 2xx response within 5 seconds. For slow processing, immediately acknowledge the webhook and handle it asynchronously using a queue.

Event payload example

json 
{
  "id":        "evt_01HXZ4KQ3T9V",
  "type":      "visit.verified",
  "createdAt": "2026-01-20T08:31:05Z",
  "data": {
    "visitId":   "vis_abc789",
    "repId":     "usr_amara",
    "companyId": "cmp_001",
    "coords":    { "lat": -1.2921, "lng": 36.8219 },
    "verified":  true
  }
}

# Rate Limits

Rate limits are applied per API key, per minute. Limits vary by plan. The current window usage is returned in every response header.

Plan Requests / min Visits / day Webhooks
Starter 60 500 2 endpoints
Growth 300 5,000 10 endpoints
Enterprise Unlimited Unlimited Unlimited
Example — Growth plan: 187 / 300 requests this minute

Rate limit headers

http response headers
X-RateLimit-Limit:     300
X-RateLimit-Remaining: 113
X-RateLimit-Reset:     1737541200   # Unix timestamp
Retry-After:           23           # seconds (only on 429)
🚫
429 Too Many Requests When rate-limited, back off exponentially starting at the Retry-After value. Repeated violations may trigger temporary IP blocks.

# Error Codes

All errors follow a consistent structure. The code field is machine-readable; message is human-readable and safe to surface in logs.

json — error response
{
  "error": {
    "code":    "GEOFENCE_VIOLATION",
    "message": "Check-in location is 840m from the registered company address.",
    "status":  422,
    "meta": {
      "distance":  840,
      "tolerance": 200
    }
  }
}
Code HTTP Meaning
UNAUTHORIZED 401 Missing or invalid API key.
FORBIDDEN 403 Key lacks required scope.
NOT_FOUND 404 Resource doesn't exist or is archived.
VALIDATION_ERROR 422 Request body failed schema validation.
GEOFENCE_VIOLATION 422 GPS coords outside company tolerance.
DUPLICATE_VISIT 409 Idempotency key already used.
RATE_LIMITED 429 Too many requests. See Retry-After header.
INTERNAL_ERROR 500 Something went wrong on our end. Retryable.

# JS / Node SDK Reference

The official @safitrack/sdk-core package wraps the REST API with typed methods, automatic retry logic, and offline queue support.

Client configuration

Option Type Default Description
apiKey string — required Your API key.
region string africa-west-1 Nearest server region.
timeout number 10000 Request timeout in ms.
retries number 3 Auto-retry attempts on 5xx.

# Python SDK

The Python SDK mirrors the JS SDK API. Install via pip and use with any async or sync framework.

bash 
pip install safitrack
python 
from safitrack import SafiTrack

safi = SafiTrack(api_key="st_live_xxxx", region="africa-west-1")

# List all verified visits from January 2026
visits = safi.visits.list(
    status="verified",
    from_date="2026-01-01",
    to_date="2026-01-31",
)

for v in visits.data:
    print(v.id, v.company_id, v.created_at)
