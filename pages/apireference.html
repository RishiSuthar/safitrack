REST API v1

API Reference

The SafiTrack REST API lets you programmatically manage field operations, sync data, and build integrations. All requests and responses use JSON.

API Status: Operational
Base URL https://api.safitrack.io/v1
Last updated Jan 20, 2026
Authentication

Authentication

SafiTrack uses API key authentication for server-to-server requests. Pass your key in the Authorization header as a Bearer token.

Authorization: Bearer st_live_xxxxxxxxxxxxxxxxxxxx
🔑
Live keys st_live_…
Used in production. Billable actions apply. Never expose in client-side code.
🧪
Test keys st_test_…
Sandbox environment. No real data affected. Safe for CI/CD pipelines.
⚡
Session tokens st_sess_…
Short-lived tokens (4h TTL) for PWA/browser use. Generated server-side via POST /auth/session.
🔄
Key rotation
Rotate keys without downtime via POST /keys/rotate. Old key stays valid for 60 seconds during rotation.
Companies

Companies

Companies are the primary entities your field team visits. Each company has a location, contacts, and an inventory of tracked products.

List all companies
Query Parameters
limit
integer
Optional — Number of results per page. Default 50, max 200.
after
string
Optional — Cursor for pagination. Use pagination.nextCursor from prior response.
territory_id
string
Optional — Filter by territory. E.g. ter_nairobi_cbd.
search
string
Optional — Fuzzy search on company name, phone, and address.
status
enum
Optional — active | archived. Defaults to active.

DELETE
/companies/:id
Archive a company
⚠️
Soft delete only. Companies are archived, not permanently deleted. All historical visits are preserved. To permanently delete, contact support.
200 OK

  "archived":   true,
  "archivedAt": "2026-01-20T15:00:00Z"

Visits

Visits are GPS-verified check-ins performed by field reps. They capture location, timestamp, and custom form data at the point of activity.

GET
/visits
List visits with filters
Query Parameters

string
Optional — Filter by field rep.
company_id
string
Optional — Filter by company.
status

Optional — pending | submitted | verified | flagged | rejected.

date
Optional — ISO 8601 start date. E.g. 2026-01-01.
to
date
Optional — ISO 8601 end date.
limit
integer
Optional — Default 50, max 200.
200 OK

POST
/visits
Log a new visit
Body Parameters
company_id
string
Required — Company being visited.
rep_id
string
Required — Field rep performing the visit.
coords
object
Required — { lat, lng, accuracy }. Accuracy in metres.

object
Optional — Custom form fields defined in your org schema.

Optional — UUID to safely retry without duplicate records.
💡

Territories

Territories

Territories define sales zones. Assign companies and field reps to territories for structured coverage and reporting.

GET
/territories
List all territories

Returns all territories for the organization. No query parameters required.

200 OK
{
  "data": [
    {
      "id":            "ter_nairobi_cbd",
      "name":          "Nairobi CBD",
      "color":         "#2f5fd0",
      "companyCount":  47,
      "assignedReps":  ["usr_amara", "usr_kwame"]
    }
  ]
}
POST
/territories
Create a territory
Body Parameters
name
string
Required — Territory display name.
polygon
array
Optional — GeoJSON coordinate ring. If omitted, territory is company-list based.
color
string
Optional — Hex color for map display.
assigned_reps
array
Optional — Array of user IDs to assign.
201 Created 
{
  "id":           "ter_westlands",
  "name":         "Westlands",
  "color":        "#34c97a",
  "companyCount": 0,
  "createdAt":    "2026-01-20T11:00:00Z"
}
POST
/territories/cluster
Auto-cluster into territories
Body Parameters
target_count
integer
Required — Number of territories to generate.
balance_by_visit_load
boolean
Optional — Weight clusters by historical visit frequency. Default false.
preview_only
boolean
Optional — Return the proposed clusters without saving. Default true.
💡
Clustering is non-destructive by default. Set preview_only: false to commit the generated territories.
200 OK
{
  "preview": true,
  "clusters": [
    {
      "suggestedName": "Cluster A",
      "companiesCount": 38,
      "centroid": { "lat": -1.2921, "lng": 36.8219 }
    }
  ],
  "commitUrl": "/v1/territories/cluster/commit/prev_xyz"
}

Webhooks

Webhooks

Register HTTP endpoints to receive real-time event notifications. Payloads are signed with HMAC-SHA256.

GET
/webhooks
List registered webhooks

Returns all active webhook subscriptions for the organization.

200 OK
{
  "data": [
    {
      "id":        "wh_001",
      "url":       "https://yourapp.com/webhooks/safi",
      "events":    ["visit.verified", "visit.flagged"],
      "active":    true,
      "createdAt": "2026-01-01T00:00:00Z"
    }
  ]
}

string

Errors

Error Codes

All errors return a consistent JSON structure. The code field is machine-readable; message is safe to surface in logs.

{
  "error": {
    "code":    "GEOFENCE_VIOLATION",
    "message": "Check-in is 840m from company location.",
    "status":  422,
    "meta":    { "distance": 840, "tolerance": 200 }
  }
}
HTTP Code Description
401 UNAUTHORIZED Missing or invalid API key.
403 FORBIDDEN Key lacks the required scope for this action.
404 NOT_FOUND Resource does not exist or has been archived.
409 DUPLICATE_VISIT Idempotency key already used for this resource.
422 VALIDATION_ERROR Request body failed schema validation.
422 GEOFENCE_VIOLATION GPS coordinates outside company tolerance radius.
429 RATE_LIMITED Too many requests. Check Retry-After header.
500 INTERNAL_ERROR Server-side failure. Safe to retry with backoff.
Versioning

API Versioning

SafiTrack versions the API via URL path. The current stable version is v1. Breaking changes are always introduced in a new version with a 6-month deprecation notice.

📌
Pin to a date-based version by sending the x-safi-version: 2025-01-01 header. This ensures additive changes (new fields, new endpoints) don't require immediate updates on your end.
